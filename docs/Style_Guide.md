# DATABASE MONA - Style Guide

Version: 1.0

---

# 1. Filosofia

Ogni dossier deve dare l'impressione di essere stato prodotto da un'organizzazione scientifica operativa.

Il lettore non deve percepire il documento come un'opera di narrativa, bensì come un fascicolo tecnico realmente archiviato.

---

# 2. Formato

Formato pagina

- A4 verticale
- 210 × 297 mm

Orientamento

- Verticale

Tutti i dossier utilizzano esclusivamente il Master Template.

---

# 3. Griglia

La griglia del Master è permanente.

Non è consentito:

- spostare sezioni
- modificare le colonne
- cambiare i margini
- alterare il footer
- modificare l'header

Le uniche variazioni ammesse riguardano il contenuto dei riquadri.

---

# 4. Gerarchia

Ogni pagina deve essere leggibile nel seguente ordine.

1. Codice dossier
2. Titolo
3. Header
4. Evidenza principale
5. Abstract
6. Evidenze
7. Ipotesi
8. Implicazioni operative
9. Fonti
10. Collegamenti
11. Note aggiuntive

---

# 5. Colori

Palette principale

Nero

Grigio

Verde scuro MONA

Carta avorio

Le annotazioni possono utilizzare:

Blu

Rosso

Nero

Colori differenti devono essere giustificati da uno specifico autore.

---

# 6. Tipografia

Il carattere deve evocare documentazione tecnica.

Titoli

- maiuscolo
- condensato
- peso elevato

Testo

- semplice
- molto leggibile
- compatto

Il documento deve privilegiare la leggibilità rispetto all'estetica.

---

# 7. Immagini

Le immagini costituiscono prove documentali.

Sono ammessi:

- ortofoto
- fotografie
- scansioni
- radiografie
- diagrammi
- grafici
- dati GIS
- immagini satellitari

Non sono ammessi:

- concept art
- illustrazioni fantasy
- immagini decorative
- sfondi ornamentali

---

# 8. Annotazioni

Le annotazioni rappresentano modifiche successive.

Regole:

- massimo tre annotazioni
- scrittura manuale
- devono sembrare aggiunte in epoche differenti
- mai coprire testo importante
- utilizzare gli spazi liberi

Ogni annotazione deve risultare credibile.

---

# 9. Autori delle annotazioni

Ogni autore mantiene il proprio stile.

Esempio:

Serial Griller

- preciso
- tecnico
- essenziale

Deg

- osservazioni pratiche
- tono informale
- occasionale ironia

Sibililla

- pochissime parole
- schizzi
- frecce

Faz-Man

- osservazioni ingegneristiche
- note organizzative
- burocrazia

---

# 10. Carteggiatura

La carta non è nuova.

Ma nemmeno antica.

Sono ammessi:

- leggere pieghe
- angoli consumati
- piccoli segni di consultazione
- timbri

Non sono ammessi:

- carta bruciata
- macchie eccessive
- effetti horror
- danni estremi

---

# 11. Footer

Il footer identifica il documento.

Deve rimanere invariato.

Contiene:

- livello di accesso
- sincronizzazione
- data di stampa
- autore
- barcode

---

# 12. Header

L'header identifica il dossier.

Contiene:

- codice
- titolo
- categoria
- revisioni
- logo MONA
- logo H.O.T.H.

L'header è la parte più riconoscibile del progetto.

---

# 13. Evidenza principale

L'area centrale rappresenta il cuore del dossier.

Ogni pagina deve contenere una sola evidenza principale.

Essa può essere:

- una mappa
- una fotografia
- un diagramma
- una scansione
- una composizione di dati

L'evidenza deve essere immediatamente comprensibile.

---

# 14. Evoluzione

Il Master può essere migliorato.

Non può essere ridisegnato.

Ogni modifica deve aumentare la qualità mantenendo riconoscibile l'identità di DATABASE MONA.
